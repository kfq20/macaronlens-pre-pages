# Macaron Lens — 产品设计说明书

> 让 Personal Agent 第一次真正走进用户的世界

---

## 一、产品定位

### 一句话定义

Macaron Lens 是一个基于摄像头的现实世界 Personal Agent，用户无需输入文字，举起手机即可获得环境感知、智能分析和主动陪伴。

### 核心理念

**从 Chat Agent 到 Living Agent**


| 维度   | 传统 Agent | Macaron Lens |
| ---- | -------- | ------------ |
| 输入方式 | 用户打字描述   | 摄像头直接感知      |
| 交互模式 | 被动应答     | 环境驱动、主动介入    |
| 输出形式 | 纯文字回复    | 动态生成的 AR 界面  |
| 记忆维度 | 记住聊天内容   | 记住个性化内容      |
| 角色定位 | 工具       | 生活伙伴         |


### 解决的核心问题

现实生活中的大多数需求不会以文字形式出现：

- 看到一块牛肉，不知道如何处理
- 面对一桌食物，想知道热量是否超标
- 看到空水杯，忘记自己多久没喝水
- 打开冰箱，不知道哪些食材即将过期

这些需求天然具有**多模态、情境化、实时性、个性化**的特点，传统文字交互无法覆盖。

本项目满足 Hackathon 条件 A（包含功能）和条件 B（高价值场景）：

### 条件 A：Macaron 3.0 功能覆盖


| 功能              | 实现方式                               |
| --------------- | ---------------------------------- |

### 条件 B：高价值生活场景

Macaron Lens 覆盖衣食住行多维度生活场景，核心逻辑一致：

> 图像输入 → 推理用户意图（结合 Persona 个人画像 + World 环境信息）→ Tool Use → **A2UI GenUI 反馈**（动态生成交互界面，而非静态文本返回）


| 场景类别     | 示例     | Living Tool Use 链路                          |
| -------- | ------ | ------------------------------------------- |
| 食 — 饮食管理 | 拍食物/菜单 | 食物识别 → 营养数据库 → 结合减脂目标 → 热量面板                |
| 食 — 烹饪指导 | 拍食材    | 食材识别 → 烹饪知识库 → 结合口味偏好 → 做法推荐                |
| 行 — 出行导航 | 拍街景/路口 | 场景识别 + GPS定位 → 天气API + 地图API → 结合目的地 → 路线指引 |
| 衣 — 穿搭建议 | 拍衣柜/衣服 | 服装识别 → 天气API + 日程API → 结合场合 → 搭配推荐          |
| 住 — 居家管理 | 拍冰箱/植物 | 物品识别 → 保质期推理/养护知识库 → 过期提醒/浇水提醒              |

### Living Bench 理念的体现

Living Bench 的核心思想是：用户的每一次 query 不止于获取信息，而是驱动 Agent 进行推理和行动的起点。Macaron Lens 的每个场景都体现了这一理念——从一张图片出发，综合推理用户意图，调用真实世界工具，最终通过 GenUI 将结果反馈到现实中。

传统图像 AI：识别图片内容 → 返回标签。结束。

Macaron Lens 体现的 Living Bench 范式：

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│ 图像输入  │ ──→ │ 意图推理  │ ──→ │ 工具编排  │ ──→ │ 现实反馈  │
│          │     │          │     │          │     │          │
│ 摄像头画面│     │ Persona  │     │ 天气API  │     │ AR 界面  │
│          │     │ + World  │     │ 地图API  │     │ + 语音   │
│          │     │ + Intent │     │ 营养DB   │     │ + Macaron│
└──────────┘     └──────────┘     └──────────┘     └──────────┘
```

**三层推理**：

1. **Persona（用户是谁）**：用户的目标、偏好、习惯、历史
  - 例：正在减脂、喜欢日料、怕冷、第一次来这个城市
2. **World（世界是什么状态）**：通过 图像/视频/live camera + Tool Use 获取的实时环境信息
  - 例：当前气温 35°C、目的地距离 2km、今天有雨、现在是午餐时间
3. **Intent（用户想干什么）**：从图像 + Persona + World 综合推断
  - 例：拍路口 + 第一次来 + 35°C = 想找一条凉快的路走

**认路场景完整示例**：

```
用户拍摄一个陌生路口
    │
    ├─ 图像理解：识别街道、路标、周围建筑
    ├─ Persona：用户第一次来这个城市，目的地是某咖啡馆
    ├─ Tool Use：
    │   ├─ GPS 定位 → 确定当前位置
    │   ├─ 地图 API → 规划路线（多条）
    │   ├─ 天气 API → 35°C，烈日
    │   └─ POI 数据 → 沿途有遮阳骑楼/地下通道
    ├─ 综合推理：选择有遮阳的路线，而非最短路线
    │
    └─ 现实反馈：
        ├─ AR 箭头叠加在实景路口，指向推荐方向
        ├─ Macaron："走这边吧，有骑楼遮阳，虽然多走两分钟但凉快很多～"
        └─ 底部卡片：预计步行 8 分钟 | 到达时间 14:12
```

这就是 Living Tool Use 和传统导航的区别：不只是规划路线，而是理解"这个人、在这个天气、在这个场景下，真正需要什么"。

## 三、核心功能模块

### 模块 1：Camera is the New Prompt（视觉感知）

**原理**：摄像头取代文字输入，成为新的 Prompt 入口。

**流程**：

```
用户举起手机 → 摄像头捕获画面 → 视觉模型识别 → Agent 理解场景 → 生成响应
```

**能力**：

- 食物识别：种类、份量、烹饪方式
- 肉类识别：部位、纹理方向、品质评估
- 文字识别：菜单、标签、成分表
- 场景理解：餐桌、厨房、办公桌等环境

---

### 模块 2：Proactive Agent（主动交互引擎）

**原理**：Agent 不再等待用户提问，而是基于环境观察主动介入。

**触发机制**：

```
环境观察 → 环境理解 → 需求预测 → 主动提醒 → 用户互动
```

**场景示例**：


| 环境信号     | Agent 行为          |
| -------- | ----------------- |
| 检测到高热量午餐 | 主动提示脂肪摄入偏高，建议晚餐调整 |
| 看到空水杯    | 温馨提醒喝水            |
| 深夜仍在工作   | 提醒已连续工作数小时        |
| 今日第三杯咖啡  | 结合记忆提醒咖啡因摄入量      |


---

### 模块 3：双工前端交互（Duplex Interaction）

**原理**：传统 Agent 交互是半双工的——用户说完，Agent 才回复。Macaron Lens 实现全双工：Agent 持续接收环境画面（上行），同时实时向用户推送动态界面（下行），两个方向并行不阻塞。

**与传统交互的区别**：


| 维度  | 传统 Agent（半双工）   | Macaron Lens（全双工） |
| --- | --------------- | ----------------- |
| 上行  | 用户打字发送 → 等待     | 摄像头持续传输画面         |
| 下行  | Agent 回复文字 → 结束 | AR 界面实时更新         |
| 节奏  | 一问一答，轮流进行       | 双向同时，持续流动         |
| 状态  | 静态：等待 or 回复     | 动态：始终在线           |


**具体表现**：

1. **持续感知（上行不断）**：摄像头画面持续输入 Agent，环境变化实时被捕获——新的食物出现、杯子被喝空、用户移动视角，Agent 都能感知
2. **实时反馈（下行不断）**：AR Overlay 随环境变化动态更新——食物移动时标签跟随、新物体进入画面时自动标注、识别结果逐步细化
3. **交互不中断**：用户点击 AR 界面上的按钮时，摄像头仍在工作；Agent 主动弹出提醒时，不影响用户继续扫描其他物品

**技术实现**：Camera + AR Overlay + GenUI 三者配合

- **Camera**：提供持续的环境输入流
- **AR Overlay**：提供实时的视觉输出层
- **GenUI**：根据识别结果动态生成 UI 组件，叠加在真实画面上

---

### 模块 4：GenUI（动态界面生成）

**原理**：Agent 输出不是纯文字，而是根据场景实时生成的可交互 UI 组件。

**特点**：

- 非预定义模板，由 Agent 根据识别结果动态生成
- AR Overlay 形式叠加在真实画面上
- 支持用户点击交互，深化双工循环

**UI 类型**：

- 悬浮标签：食物上的热量/营养标注
- 信息卡片：详细的营养分析面板
- 操作指引：肉类切割方向箭头
- 推荐列表：菜品推荐组合

---

### 模块 4：Living Tool Use（活的工具调用）

**原理**：工具调用由环境驱动，而非用户指令驱动。

**与传统 Tool Use 的区别**：

```
传统：用户提问 → 调用工具 → 返回结果

Macaron Lens：环境观察 → 环境推理 → 规划工具 → 调用工具 → 现实反馈
```

**工具链示例**：

- 拍摄食物 → 食物识别 API → 营养数据库查询 → 热量计算 → 生成饮食面板
- 拍摄牛肉 → 部位识别 → 烹饪知识库 → 生成切割指引和做法推荐
- 拍摄菜单 → OCR 识别 → 营养分析 → 生成点菜建议

---

### 模块 5：Reality-grounded Memory（现实记忆）

**原理**：Agent 记住的不是聊天记录，而是用户的生活习惯和偏好。

**记忆维度**：

- 健康目标：正在减脂、每日热量目标
- 饮食偏好：喜欢的食物、过敏信息
- 生活习惯：咖啡频率、饮水习惯、作息规律
- 历史数据：每日摄入记录、运动数据

**记忆应用**：
当再次出现相关场景时，Agent 结合长期记忆进行个性化推理，提供更精准的建议。

## 五、Macaron 角色设计

### 角色定位

Macaron 不是一个冰冷的分析工具，而是一个有温度、可感知、可交互的 Living Companion。它是双工交互体验的核心载体——用户与 Agent 之间所有的情感连接，都通过 Macaron 这个角色来承载。

### 形象设定

- 视觉形态：小巧可爱的 AR 虚拟角色
- 出现方式：坐在杯子上、趴在书本上、站在餐桌边
- 性格特征：温暖、俏皮、关心用户但不打扰

### 核心交互能力

#### 1. 位置感知 — Macaron 知道自己在哪

Macaron 不是随机浮在屏幕上的气泡，它能感知自己"贴"在现实世界的哪个物体上，并据此产生对应行为：


| Macaron 所在位置 | 感知行为        |
| ------------ | ----------- |
| 贴在水杯上        | 感知杯中水量，提醒喝水 |
| 贴在餐盘旁        | 感知食物内容，分析热量 |
| 贴在书本上        | 感知学习状态，鼓励用户 |
| 贴在电脑旁        | 感知工作时长，提醒休息 |
| 贴在菜单上        | 感知菜品列表，推荐搭配 |


位置不同，Macaron 的"关注点"也不同——它真正理解自己身处的语境。

#### 2. 可拖动 — 用户决定 Macaron 关注什么

用户可以用手指把 Macaron 从一个位置拖到另一个位置：

**交互流程**：

```
用户长按 Macaron → 拖动到新位置 → Macaron 感知新环境 → 产生新的行为
```

**示例**：

- 把 Macaron 从桌面拖到水杯上 → 它开始关注你的饮水
- 把 Macaron 从水杯拖到餐盘旁 → 它开始分析你的饮食
- 把 Macaron 拖到手机角落 → 它安静待着，不主动打扰

拖动本身就是一种无需语言的指令——用户通过改变 Macaron 的位置，隐式地告诉它"我现在关心什么"。

#### 3. 可对话 — 点击唤醒，语音/文字互动

用户可以随时点击 Macaron 与它对话：


| 交互方式   | Macaron 反应            |
| ------ | --------------------- |
| 单击（轻点） | Macaron 转向用户，弹出文字气泡回应 |
| 双击     | 展开详细信息面板（如完整营养报告）     |
| 长按     | 进入拖动模式                |
| 对着它说话  | 语音对话，Macaron 口头回应     |


**点击反馈示例**：

- 无事发生时点它："嗯？怎么啦～有什么需要帮忙的吗？"
- 它正在分析食物时点它："等一下哦，我在看你吃了什么～"
- 已经闲置很久时点它："好久没见你啦，最近在忙什么呀？"

#### 4. 情绪反馈 — Macaron 有自己的状态

Macaron 的外观和动作会随场景变化：


| 状态           | 表现        |
| ------------ | --------- |
| 开心（用户完成目标）   | 蹦跳、冒小星星   |
| 担心（检测到不健康行为） | 歪头、冒问号    |
| 专注（正在分析）     | 眯眼、转圈思考   |
| 犯困（深夜陪伴）     | 打哈欠、揉眼睛   |
| 被拖动时         | 惊讶表情、晃动小手 |
| 被点击时         | 转头看向用户、眨眼 |


### 交互语言风格


| 场景      | Macaron 的话              |
| ------- | ----------------------- |
| 看到空杯子   | "啊偶～杯子里没有水了诶，要不要喝一点水呢？" |
| 用户完成目标  | "太棒啦！今天已经喝够水啦！"         |
| 看到高热量食物 | "今天脂肪摄入有点高哦～"           |
| 看到用户学习  | "又进步了一点点呢～"             |
| 深夜工作    | "已经连续工作三个小时啦，休息一下吧～"    |
| 被拖到新位置  | "哦？要我看看这个吗～"            |
| 被闲置后点击  | "嘿嘿，想我了吗？"              |
| 用户对它说谢谢 | "不客气呀，这是我应该做的～"         |


### 设计原则

- **有感知**：知道自己在哪、看到了什么，行为有据可依
- **可操控**：用户通过拖动和点击来引导 Macaron 的关注点
- **有温度**：语气亲切自然，像朋友而非助手
- **有状态**：情绪和动作随场景变化，感觉是"活的"
- **不打扰**：只在合适的时机主动开口，被点击时才回应闲聊
- **有记忆**：记住用户的习惯，越用越懂你
- **可分享**：互动瞬间天然适合社交分享

## 六、社交传播设计

### Living Moments

每一次互动都可以生成可分享的"生活瞬间"：

```
现实世界 × AR Overlay × Macaron × 个性化故事 = 天然 UGC 内容
```

### 传播逻辑

用户不会分享"AI 分析了热量"，但会分享：

- "哈哈，它今天又爬到我的杯子上了"
- "它说我今天红烧肉吃太多了"
- "Macaron 给我推荐的午餐只有 550 卡"

AI 最大的传播媒介不是聊天记录，而是**生活本身**。
